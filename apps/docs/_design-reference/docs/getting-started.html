Getting started

Take an existing Next.js app from next build to a scale-to-zero Knative service in about five minutes. knext uses the official Next.js Deployment Adapter API — no fork, no custom runtime.

Prerequisites. A Kubernetes cluster with Knative Serving installed, Next.js 16.2+, and a container registry. New to Knative? The operator-managed install sets up Serving and the correct ingress class for you.

1 · Point Next.js at the adapter

Install knext and reference the adapter from your next.config.ts. The adapter forces output:'standalone' at build time.

  next.config.ts
import path from 'node:path';
import type { NextConfig } from 'next';

const config: NextConfig = {
  output: 'standalone',
  experimental: {
    // knext ships the official NextAdapter as a local adapter file.
    adapterPath: path.resolve(import.meta.dirname, 'next-adapter.ts'),
  },
};
export default config;

2 · Build, push, declare

Build a digest-pinned image, then describe the deployment as a NextApp custom resource. The CLI emits the CR; the operator does the rest.

  shell
# build + push (digest is resolved automatically)
$ npx kn-next deploy --registry registry.example.com/storefront

# the CLI emits a NextApp CR; the operator reconciles it
✓ image  registry.example.com/storefront@sha256:9f1c…
✓ NextApp/storefront applied
✓ url    https://storefront.apps.example.com  (scaled to 0)

The operator is the source of truth

knext never mutates the cluster out-of-band. The Go operator reconciles your NextApp into a Knative Service — wiring autoscaling, the bytecode-cache volume, and digest-pinned images. The CLI only ever applies the CR.

Security by default. The operator rejects mutable tags (:latest) — every image is pinned by @sha256:. Backends are cluster-local; mutating endpoints require auth.

Bytecode-cached cold starts

Scale-to-zero means cold starts — so knext sets NODE_COMPILE_CACHE to a persistent volume. On the first start V8 serializes compiled bytecode to disk; every later cold pod reads it and skips JS parsing. It's the self-hosted version of the mechanism behind Vercel Fluid.

  nextapp.yaml — cache
spec:
  cache:
    enableBytecodeCache: true   # mount a PVC, set NODE_COMPILE_CACHE
  scaling:
    minScale: 0        # idle → zero pods
